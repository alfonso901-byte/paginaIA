Tema 3.6: Arquitectura de un Sistema de Producción (SBR)

1. Nombre del tema

Arquitectura de un Sistema de Producción: Hechos y Base de Conocimientos

2. Fundamento teórico

Un Sistema de Producción o Sistema Basado en Reglas (SBR) organiza el conocimiento experto en dos repositorios diferenciados:

Hechos (Working Memory)

  • Definición: Conjunto dinámico de hechos que describen el estado actual del dominio. Cada hecho puede representarse como un par atributo–valor, una tupla estructurada o incluso como un grafo de relaciones.
  • Representación:
    • Hechos atómicos: (temperatura 75°F), (sensor activo true).
    • Hechos compuestos: estructuras jerárquicas o redes semánticas.
  • Gestión: el motor de inferencia puede assert (añadir), retract (retirar) o modify (modificar) hechos en cada ciclo.

Base de Conocimientos (Rule Base)

  • Definición: Colección de reglas de producción en formato IF–THEN que codifican el conocimiento experto.
  • Estructura de una regla:
    • Antecedente (condiciones): conjunto de patrones de hechos que deben cumplirse.
    • Consecuente (acciones): operaciones sobre la Working Memory (añadir, retirar, modificar hechos).
  • Clasificación:
    • Reglas de inferencia: generan nuevos hechos.
    • Reglas de validación: verifican consistencia.
    • Reglas de control: gestionan flujo de activación.
  • Mantenimiento: la modularidad permite actualizar o ampliar el Rule Base sin afectar el motor.

Ciclo de Inferencia

  1. Matching: comparador de hechos vs. condiciones de reglas.
  2. Conflict resolution: política para seleccionar una regla entre varias disparables (prioridad, recencia, especificidad).
  3. Firing: ejecución de la regla y actualización de la Working Memory.
  4. Repetición: hasta que no haya reglas aplicables o se alcance un objetivo.

Optimización con Rete

El algoritmo Rete organiza las condiciones de todas las reglas en una red de nodos que comparten pruebas parciales, minimizando la reevaluación completa de hechos y acelerando el matching en sistemas de gran escala.

5. Conclusión

La clara separación entre hechos y reglas, junto con un ciclo de inferencia optimizado, permite construir sistemas expertos modulares, transparentes y capaces de escalar en complejidad, siempre que se gestionen adecuadamente las políticas de resolución de conflictos.
