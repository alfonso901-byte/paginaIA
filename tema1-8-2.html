Tema 1.8.2: Algoritmo A*

1. Nombre del tema

Algoritmo A*

2. Fundamento teórico

A* es un algoritmo de búsqueda informada que encuentra caminos óptimos en un grafo ponderado combinando el coste real desde el inicio y una heurística que estima el coste restante.

Definición de la función de evaluación:

  • g(n): coste acumulado desde el nodo inicial hasta n.
  • h(n): heurística admisible que estima el coste desde n hasta el objetivo. Admisible significa que nunca sobreestima el coste real.
  • f(n) = g(n) + h(n): ordena los nodos en la frontera, priorizando los más prometedores.

Propiedades clave:

  • Optimalidad: A* garantiza encontrar el camino de coste mínimo si h(n) es admisible y consistente (monótona): h(n) ≤ cost(n, m) + h(m) para cualquier sucesor m de n.
  • Completitud: encuentra una solución si existe un camino finito al objetivo.
  • Complejidad: en peor caso exponencial, aunque eficiente con heurísticas informativas.

Implementación básica:

initialize open_set with start node
initialize closed_set empty
while open_set not empty:
  current = node in open_set with lowest f(n)
  if current is goal: return path
  move current from open_set to closed_set
  for each neighbor of current:
    tentative_g = g(current) + cost(current, neighbor)
    if neighbor in closed_set and tentative_g ≥ g(neighbor):
      continue
    if neighbor not in open_set or tentative_g < g(neighbor):
      parent[neighbor] = current
      g[neighbor] = tentative_g
      h[neighbor] = heuristic(neighbor)
      f[neighbor] = g[neighbor] + h[neighbor]
      if neighbor not in open_set: add neighbor

Elección de heurísticas:

  • Distancia Euclidiana: para entornos continuos sin obstáculos.
  • Distancia Manhattan: para movimientos en cuadrícula ortogonal.
  • Distancia Chebyshev: para cuadrículas con movimiento diagonal.

5. Conclusión

El algoritmo A* combina eficacia y optimalidad gracias a su función de evaluación que integra coste real e heurística. Su éxito depende de elegir heurísticas que sean admisibles y consistentes.
