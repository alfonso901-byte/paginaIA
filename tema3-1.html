Tema 3.1: Representación de Conocimiento mediante Reglas

1. Nombre del tema

Representación de Conocimiento mediante Reglas

2. Fundamento teórico

Las reglas de producción son cláusulas del tipo IF condición THEN acción que permiten codificar conocimiento declarativo y procedimental en sistemas basados en reglas.

Componentes de una regla

Antecedente (LHS)

Conjunto de hechos o condiciones que deben cumplirse.

Consecuente (RHS)

Acción o conclusión que se deduce cuando el antecedente es verdadero.

Tipos de reglas

  • Reglas definitorias: definen conceptos o clases.
  • Reglas de inferencia: generan nuevos hechos a partir de hechos existentes.
  • Reglas de restricción: validan consistencia de datos.

Arquitectura de un sistema basado en reglas

  • Base de Conocimiento: hechos y reglas.
  • Motor de Inferencia: aplica reglas (forward/backward chaining).
  • Agenda: lista de reglas activadas pendientes.
  • Control de Conflictos: estrategia para seleccionar qué regla disparar.

Representación del conocimiento

  • Permiten modularizar y localizar conocimiento en unidades lógicas.
  • Diferencian de forma clara el conocimiento declarativo (el qué) y el procedimental (el cómo).

Aspectos metodológicos

  • Talleres con expertos para elicitar reglas (Knowledge Engineering).
  • Ciclos iterativos: diseño, validación y refinamiento de reglas.
  • Herramientas: Drools, CLIPS, Jess para desarrollo y mantenimiento.
