Tema 2.7: Conocimiento No-Monótono y Otras Lógicas

1. Nombre del tema

Conocimiento No-Monótono y Otras Lógicas

2. Fundamento teórico

A diferencia de la lógica monótona, el razonamiento no-monótono permite retractar conclusiones cuando se añade información nueva o se descubren excepciones. Es esencial para modelar conocimiento real, donde las reglas suelen tener excepciones.

Principios clave

  • Defeasibilidad: las conclusiones pueden quedar invalidadas por nueva evidencia.
  • Orden de preferencia: las reglas poseen prioridades para resolver conflictos.
  • Contexto cerrado: lo no demostrado como verdadero se asume falso (CWA).

Principales sistemas

  • Logic Programming with Defaults (LPwD): extiende Prolog con defaults y excepciones.
  • Default Logic (Reiter 1980): reglas por defecto α:β/γ que se aplican salvo contradicción.
  • Autoepistemic Logic (Moore 1985): permite razonar sobre las propias creencias.
  • Circumscription: minimiza la extensión de predicados para asumir sólo lo necesario.

Representación del conocimiento

  • Reglas por defecto y excepciones explícitas.
  • Integración con ontologías y OWL RL.

Mecanismos de inferencia

  • Revisión iterativa de conclusiones ante nuevas premisas.
  • Resolución de conflictos mediante prioridades de reglas.

Aspectos metodológicos

  • Talleres con expertos para identificar excepciones y prioridades.
  • Ciclos de refinamiento: prueba, detección de excepciones y actualización.
  • Herramientas: XSB-Prolog (tabling) y DLV (Answer Set Programming).
